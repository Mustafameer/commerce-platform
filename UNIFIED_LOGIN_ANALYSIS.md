### 2. صفحة الدخول (موحدة):
```
┌─────────────────────────┐
│   تسجيل الدخول         │
├─────────────────────────┤
│ 📱 الهاتف              │
│ [________________]      │
│                         │
│ 🔐 كلمة المرور        │
│ [________________]      │
│                         │
│ [   دخول   ]           │
└─────────────────────────┘
```

### 3. API Endpoint (موحد):
```
POST /api/login
{
  "phone": "07810909577",
  "password": "123456"
}

الرد (يتضمن store_type):
{
  "id": 5,
  "name": "علي الهادي",
  "phone": "07810909577",
  "role": "merchant",
  "store_id": 13,
  "store_type": "topup",      ← يتم التوجيه بناءً على هذا
  "store_active": true,
  "store_status": "approved"
}
```

### 4. التوجيه في الواجهة الأمامية:
```javascript
if (user.store_type === 'topup') {
  navigate('/topup-merchant');  // منصة متاجر الشحن
} else {
  navigate('/merchant');        // منصة المتاجر العادية
}
```

## 🎯 الفائدة: بنية موحدة بالكامل

| الجانب | التفصيل |
|-------|---------|
| 📱 **الدخول** | موحد (الهاتف + كلمة المرور) ✅ |
| 🔐 **كلمة المرور** | مشتركة (نفس الطريقة) ✅ |
| 📞 **الهاتف** | مشترك (نفس التنسيق) ✅ |
| 💾 **قاعدة البيانات** | موحدة (جدول users واحد) ✅ |
| 🔀 **التوجيه** | تلقائي حسب store_type ✅ |
| 🎨 **الواجهات** | منفصلة ولكن بدخول موحد ✅ |
