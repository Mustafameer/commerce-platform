# نظام التحقق من العملاء في متاجر الشحن

## 📋 الوصف العام

تم تطبيق ميزة التحقق من العملاء في شاشة استكشاف المتاجر. عند الضغط على متجر بطاقات شحن، يظهر نموذج طلب اسم العميل ورقم هاتفه للتحقق قبل الدخول إلى المتجر.

---

## 🎯 المتطلبات المحققة

✅ عند الضغط على متجر بطاقات شحن → يظهر نموذج تحقق  
✅ النموذج يطلب: اسم العميل + رقم الهاتف  
✅ التحقق من البيانات في قاعدة البيانات  
✅ الدخول إلى متجر الشحن بعد التحقق الناجح  
✅ رسالة خطأ واضحة في حالة عدم تطابق البيانات  

---

## 🔧 الميزات التقنية المضافة

### 1. دالة تطبيع أرقام الهاتف
```javascript
// في StoresPage (src/App.tsx)
const normalizePhone = (phone: string): string => {
  // تحويل الأرقام إلى تنسيق موحد
  // يدعم الصيغ المختلفة: 07800000000, 9647800000000, +9647800000000
}
```

### 2. نموذج التحقق
- حقل اسم العميل (نص عادي)
- حقل رقم الهاتف (رقم)
- زر تحقق (التحقق من البيانات)
- زر إلغاء (إغلاق النموذج)
- رسائل خطأ واضحة بالعربية

### 3. عملية التحقق
```
1. يجلب النموذج قائمة العملاء من API
2. يقارن اسم العميل ورقم الهاتف مع البيانات المسجلة
3. إذا تطابقت البيانات → ينقل إلى صفحة متجر الشحن
4. إذا لم تتطابق → يعرض رسالة خطأ
```

- **المسار**: `/api/merchant/customers`

- **التحسين**: إضافة معالجة الأخطاء

## 🗄️ جدول قاعدة البيانات

```sql
CREATE TABLE IF NOT EXISTS customers (
    id SERIAL PRIMARY KEY,
    store_id INTEGER NOT NULL REFERENCES stores(id),
    name VARCHAR(255) NOT NULL,
    phone VARCHAR(20) NOT NULL,
    email VARCHAR(100),
    customer_type VARCHAR(50) DEFAULT 'cash',
    credit_limit DECIMAL(10, 2) DEFAULT 0,
    current_debt DECIMAL(10, 2) DEFAULT 0,
    notes TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(store_id, phone)
);
```

## 📝 كيفية إضافة عملاء لمتجر الشحن

### الطريقة 1: من لوحة تحكم التاجر
تأكد من أن لديك صفحة إدارة العملاء في لوحة التاجر حيث يمكنه:
1. إضافة عميل جديد (اسم + هاتف)
2. تعديل بيانات عميل موجود
3. تفعيل/تعطيل حساب العميل

### الطريقة 2: SQL (إدراج مباشر)
```sql
INSERT INTO customers (store_id, name, phone, customer_type, is_active)
VALUES (1, 'محمد علي', '07800000000', 'cash', true);
```

### الطريقة 3: API
```javascript
POST /api/customers
{
  "store_id": 1,
  "name": "محمد علي",
  "phone": "07800000000",
  "customer_type": "cash",
  "email": "customer@example.com"
}
```

## 🐛 استكشاف الأخطاء

### المشكلة: العميل غير قادر على الدخول
**الحل**:
1. أكد من أن اسم العميل مطابق تماماً (بما فيه المسافات)
2. تحقق من رقم الهاتف (جرب الصيغ المختلفة)
3. تأكد من أن العميل مفعّل (`is_active = true`)

### المشكلة: يظهر الخطأ مع بيانات صحيحة
**الحل**:
1. افتح أدوات المطور (F12)
2. افحص الطلب في Network tab
3. تحقق من البيانات المُرجعة من API

### المشكلة: النموذج لم يظهر
**الحل**:
1. تأكد من أن `store.store_type === 'topup'`
2. افحص وحدة التحكم (Console) للأخطاء

## 📊 مثال عملي كامل

### 1. إضافة عميل
```sql
INSERT INTO customers (store_id, name, phone, is_active)
VALUES (5, 'أحمد محمد', '07812345678', true);
```

### 2. محاولة الدخول
- انتقل إلى صفحة استكشاف المتاجر
- اضغط على متجر "كارتات شحن"
- أدخل: 
  - الاسم: `أحمد محمد`
  - الهاتف: `07812345678` أو `9647812345678`
- اضغط "تحقق الآن"
- سينقلك مباشرة إلى المتجر ✅
